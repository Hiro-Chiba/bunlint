# ユーザーが対応する残作業まとめ

- Gemini APIキーを取得し、`.env`とVercelプロジェクトの環境変数に`GEMINI_API_KEY`として設定する。
- NeonでPostgreSQLインスタンスを用意し、接続文字列を`.env`の`DATABASE_URL`とVercel環境変数に登録する。
- Vercelプロジェクトを作成し、GitHubリポジトリと連携させたうえで、デプロイトリガーを確認する。
- 必要に応じてNeon側でIP制限や接続プール設定を行い、運用時の安定性を確保する。
- Gemini APIとVercelの利用料金・クォータを確認し、想定するトラフィックに合わせて上限や警告を設定する。
- 本番公開前に、環境変数が正しく読み込まれているか、Vercelプレビュー環境で最終確認する。
